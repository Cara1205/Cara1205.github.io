从删库到跑路By张馨月

——MySQL建库删库之旅

MySQL

在官网中点击DOWNLOADS—Community，下载社区版MySQL8.0.12。

安装完成

然后准备开始建数据库

先看一看技术部的过程要求

虽然不是太懂，不过我刚刚下载下来的东西里有一个就叫MySQLWorkbench，不能用这个建表就说明有伏笔。

数据库的建立

先定一个小目标，按照教程建立一个CMS的数据库吧

如何用Mysql的命令行客户端创建数据库

（未经许可转载，作者可联系我删除）

好像是版本与语法不对应，而网上的教程大多是5.6版本的，决定把当前版本卸载重装。（重装后是8.0.11版）

这一次，我更换了一个教程，因为之前无论怎么试都失败了（大家可以看看我上一张图里，看看错误在哪儿。这么低级的错误一定不要犯）

#

mysql通过命令创建数据库，创建并操作表

这个教程里包含了创造数据库以及操作表的详细步骤，简单易读，也没有使用MySQLWorkbench。推荐（未经许可引用，作者可联系我删除）

可以看到，我第一次还是输错了，第二次照着教程走，一次成功。

#

创建数据库表

用户信息表

使用代码

create table "name" (sorts char());

创建成功后可以查看一下是否创建成功、数据表格式如何

show tables;
describe "name";

在这里换行时忘记查看教程，以为是因为没有输入“;”导致了第一次的错误。改正错误后输入数据成功。

输入了乔布斯和神座出流的数据（不打码了，为什么我上一张要打码）~因为出流的出生年未知，我本打算用xxxx代替，不过MySQL在这里是严格按照日期审核的，不允许输入xxxx。

随后尝试了一下教程里的用文件一次性输入，不过失败了。（等我把这篇博客写完再去找原因）

#

文章信息表

为了做这一板块，参考了一下第一次博客中自带的文章属性分类
，分为layout、 title、 date、 tags


（两次见证我的手癌与脑癌）

同样按照教程中的方式，创建好文章信息表后，开始导入信息。企图用一些中文的文章名，不可以；企图在日期后加上时间点，不可以；文章分类用汉字，不可以。

修正过后，文章信息也成功导入啦。

#

用户权限

为了做这一板块，百度了一下，应该具备的是用户名、角色和权限。

其中character可能与MySQL中的语言重复了，不允许这样使用。

然后上传用户信息，建表完成~

#

#尾声——跑路#

重头戏终于来了

先来看看我们创建好的数据库中的表格，一共有用户信息、文章信息以及用户权限三个表格。

使用代码

drop database baidu;

检查一下是否删除成功

use baidu;

Hexo博客搭建过程by张馨月

前言

技术部试题：此为本人的Hexo博客的搭建过程，耗时超过6个小时，从typecho到华为云Wordpress再到Hexo，一路波折终于搭建成功啦。简单发图片回顾一下过程(注：有参考许多教程，Windows系统)

过程

#

  • typecho

    试着安装了PHP与SQLite,发现自己不会安装扩展，无奈放弃。

.

  • WordPress

    在网上搜到了一个华为云WordPress，注册并实名认证后发觉大事不妙(?!)，这不像是个博客反倒像个交易平台啊，于是赶紧换到了hexo

    （第二天有北京客服给我打电话询问我注册是否需要什么服务，请慎重对待）

hexo

hexo 的官网内有大部分的安装教程。

引用自hexo官网

此后才是正戏（误

  • 参照了几份教程（未经允许不能把人家的网址发上来，大家自己搜索一下吧）。

在部分教程中代码为：

$ npm install hexo-cli -g

为了防止后续可能出现的错误，接着运行：

$ npm install hexo-deployer-git --save

接下来回到hexo文件夹（这个文件夹是自己建的，里面是空的，不是直接从网上下载到安装包安装的），鼠标右键，选择Git Bash，执行以下命令，Hexo会自动在该文件夹下下载搭建网站所需的所有文件。

顺便求解一下这个问题，因为我安装了过后看见教程说最好不要在中文路径下，就把安装文件改成了Installation Documents,中途遇见bug，把Git直接放进了回收站里，再重新运行安装包（为了省流量没有重新下载），发现还是安装在中文路径下，不过仍然能使用。

本地浏览：http://localhost:4000/

注册Github账号

接着注册一个Github的账号，进入https://github.com/进行注册，用户名、邮箱和密码之后都需要用到，一定要记好。

创建repository

如图，点击New创建，名称格式一定要为yourname.github.io
注意：此处的yourname必须为你的github用户名，用户名是可以用来登陆的。

敲重点！！一定记得在冒号后面打上一个空格！！

是不是看上去很容易？别忘了还有一个ssh文件的配置。大概操作就是：

  1. 任意地方鼠标右键，打开Git Bash
$ ssh-keygen -t rsa -C "your_email@youremail.com"

把示例中的邮箱改为在Github注册的邮箱，然后一路回车。（有出现一串方框样式的代码就成功啦）
