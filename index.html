Hexo博客搭建过程by张馨月

前言

技术部试题：此为本人的Hexo博客的搭建过程，耗时超过6个小时，从typecho到华为云Wordpress再到Hexo，一路波折终于搭建成功啦。简单发图片回顾一下过程(注：有参考许多教程，Windows系统)

过程

#

  • typecho

    试着安装了PHP与SQLite,发现自己不会安装扩展，无奈放弃。

.

  • WordPress

    在网上搜到了一个华为云WordPress，注册并实名认证后发觉大事不妙(?!)，这不像是个博客反倒像个交易平台啊，于是赶紧换到了hexo

    （第二天有北京客服给我打电话询问我注册是否需要什么服务，请慎重对待）

hexo

hexo 的官网内有大部分的安装教程。

引用自hexo官网

此后才是正戏（误

  • 参照了几份教程（未经允许不能把人家的网址发上来，大家自己搜索一下吧）。

在部分教程中代码为：

$ npm install hexo-cli -g

为了防止后续可能出现的错误，接着运行：

$ npm install hexo-deployer-git --save

接下来回到hexo文件夹（这个文件夹是自己建的，里面是空的，不是直接从网上下载到安装包安装的），鼠标右键，选择Git Bash，执行以下命令，Hexo会自动在该文件夹下下载搭建网站所需的所有文件。

顺便求解一下这个问题，因为我安装了过后看见教程说最好不要在中文路径下，就把安装文件改成了Installation Documents,中途遇见bug，把Git直接放进了回收站里，再重新运行安装包（为了省流量没有重新下载），发现还是安装在中文路径下，不过仍然能使用。

本地浏览：http://localhost:4000/

注册Github账号

接着注册一个Github的账号，进入https://github.com/进行注册，用户名、邮箱和密码之后都需要用到，一定要记好。

创建repository

如图，点击New创建，名称格式一定要为yourname.github.io
注意：此处的yourname必须为你的github用户名，用户名是可以用来登陆的。

敲重点！！一定记得在冒号后面打上一个空格！！

是不是看上去很容易？别忘了还有一个ssh文件的配置。大概操作就是：

  1. 任意地方鼠标右键，打开Git Bash
$ ssh-keygen -t rsa -C "your_email@youremail.com"

把示例中的邮箱改为在Github注册的邮箱，然后一路回车。（有出现一串方框样式的代码就成功啦）

到此我的hexo博客就搭建成功啦，成功打开自己的主页的这一刻还是很开心的，不过接下来编辑第一篇文章还是有一点难度（因为不像微博什么的直接在现有的页面上编辑），需要创建新的文件，并且下载文本编辑软件进行内容的编写。

$hexo n"Hexo博客搭载过程by张馨月"

然后下载MarkdownPad

此后会报错，根据弹框提示下载一个awesomium就可以开始编写你的文章啦！
