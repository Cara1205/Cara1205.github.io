MySQL

在官网中点击DOWNLOADS—Community，下载社区版MySQL8.0.12。

安装完成

然后准备开始建数据库

先看一看技术部的过程要求

虽然不是太懂，不过我刚刚下载下来的东西里有一个就叫MySQLWorkbench，不能用这个建表就说明有伏笔。

数据库的建立

先定一个小目标，按照教程建立一个CMS的数据库吧

如何用Mysql的命令行客户端创建数据库

好像是版本与语法不对应，而网上的教程大多是5.6版本的，决定把当前版本卸载重装。（重装后是8.0.11版）

这一次，我更换了一个教程，因为之前无论怎么试都失败了（大家可以看看我上一张图里，看看错误在哪儿。这么低级的错误一定不要犯）

可以看到，我第一次还是输错了，第二次照着教程走，一次成功。

#

创建数据库表

用户信息表

使用代码

create table "name" (sorts char());

创建成功后可以查看一下是否创建成功、数据表格式如何

show tables;
describe "name";

在这里换行时忘记查看教程，以为是因为没有输入“;”导致了第一次的错误。改正错误后输入数据成功。

输入了乔布斯和神座出流的数据（不打码了，为什么我上一张要打码）~因为出流的出生年未知，我本打算用xxxx代替，不过MySQL在这里是严格按照日期审核的，不允许输入xxxx。

随后尝试了一下教程里的用文件一次性输入，不过失败了。（等我把这篇博客写完再去找原因）

为了做这一板块，参考了一下第一次博客中自带的文章属性分类

（两次见证我的手癌与脑癌）

同样按照教程中的方式，创建好文章信息表后，开始导入信息。企图用一些中文的文章名，不可以；企图在日期后加上时间点，不可以；文章分类用汉字，不可以。

修正过后，文章信息也成功导入啦。

#尾声——跑路#

重头戏终于来了

先来看看我们创建好的数据库中的表格，一共有用户信息、文章信息以及用户权限三个表格。
